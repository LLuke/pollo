Pollo, the XML editor

What is it?

Pollo lets you edit XML files. It's best suited for files which can be represented in a tree structure. It comes with built-in support for Cocoon sitemap files, Ant build files and XML Schemas.

The editor has a colourfull tree widget (not the default Swing one!), drag-and-drop support, unlimmited undo, model-view design (open multiple views on the same file), ...

Screenshots: editing an Ant buildfile, editing sitemap file , editing an xml schema.

Feature list

  • Cross-platform: Pollo is based on Java 1.3, which is available for most Windows variants, Linux, and other operating systems.
  • In contrast to many other Java-based tree XML editors, it uses its own custom-implemented widget to display XML files.
  • Supports drag-and-drop of nodes.
  • Is internally completely based on an MVC-design, the model is a standard-compliant DOM tree (Apache's Xerces implementation) that is connected to the view using standard DOM mutation events. As a consequence, if the DOM is changed by any code, the corresponding view(s) update automatically.
  • NEW Editing of any well-formed XML file: Pollo can open any well-formed XML file and display it, without needing a schema or style sheet.
  • NEW Use XPath expressions to find nodes. Pollo can execute any XPath expression, making it usefull as an XPath experimenting and learning tool.
  • Pollo can be customised for a particular XML file type by using schemas, display specifications (also known as 'style sheets'), and plugins. Pollo comes with examples for Cocoon Sitemaps, Ant buildfiles and W3C XML Schemas. For the (or a) next release, documentation will be provided on how you can add your own.
  • Schemas: the editing of a certain type of XML file is guided by a schema. A schema knows about which attributes and which subelements an element can have. It can also define picklists for attributes. NEW A powerfull feature of schema's in Pollo is that they can be chained: this way, multiple schema's can be combined as they were one big schema. This is for example usefull for combining a WSDL schema with an XML Schemas schema. The schema implementation classes and factories are defined in a configuration file, which allows for easy future extension.
  • Display specifications ('style sheets'): these tell Pollo what colors to use, and in what order the attributes must be shown on the element. Just like schemas, display specifications can be chained. The implementation classes and factories are also defined in a configuration file.
  • NEW Attribute editor plugins: it is possible to define plugins that provide the component to be used for editing an attribute value. As an example, Pollo includes a plugin for Ant that adds a button for quick access to properties. Note: in the future, other types of plugins will also be provided.
  • NEW Templates: new files can be made based on templates. In Pollo, a template is defined by a Java class, meaning that any Java code that can generate a DOM-tree can be used to generate templates on the fly (eg by a wizard that asks questions, or by reading something from a database, ...). Currently, Pollo comes with an implementation that parses a template file. Documentation on how to make your own is planned for the near future.
  • It is licensed under the terms of the MIT license, which means it is completely free for personal and comerical use.

What's the status?

Usable but not finished.

Platforms?

Pollo is written for Java 1.3. I mostly developed it on Linux but have done some testing on Windows also. Make sure you have a recent JDK version (but not 1.4).

How do I get it?

Download the zip at the sourceforge project site. Unzip the file (if you don't have a zip tool you can do this with java's jar command: java xvf pollo-0.3-bin.zip)
Pollo can then be started by executing the following command: java -jar pollo.jar
See the included README for more details.
